[WAS process]
- server tcp/ip connection wait, socket connection
- HTTP request message parsing and read
- POST method, /save URL check
- Content-Type check
- HTTP meesage body parsing
  - username, age parameter extraction
- business logic start
  - db save
# business logic end
- HTTP response message create start
    - HTTP start line create
    - HTTP header create
    - HTTP body create
- TCP/IP response send, socket connection close

[servlet]
- except business logic, servlet handle all request, response
- easy use request, response by HTTPServlet

HTTP request, response flow
- by HTTP request
  - WAS generate Request, Response instance
  - call Servlet with Request, Response instance
  - WAS generate HTTP response message by Response instance

Servlet
(Servlet Container)
- Servlet Container means WAS support servlet. ex. tomcat
- Servlet Container manage Servlet instance. generate, init, call, destroy
- Servlet instance singleton
  - Servlet Container loading time create Servlet instance
  - destroy when Servlet Container stop  
  - cause share field
- JSP -> Servlet
- support multi thread for concurrent request
